142 /*!
143  \class hum::Game
144  \brief The class that runs the Game.
145 
146  The Game class handles the game loop and the lifecycle of the game. To start
147  running a Game just call `Game::run()` and the Game instance will enter the
148  following flow chart. The flow chart shows the lifecycle of a Game and the
149  different steps that happen through it.
150 
151  \image html game_loop.png
152  \image latex game_loop.png "Game loop"
153 
154  A Game instance can be queried for Time information at any step (when the
155  Game is running). This information can be the deltaTime(), which is the Time
156  that passed between the previous `update` and the current `update`; the fixedUpdateTime() which is
157  the Time that passes between `fixedUpdate` and `fixedUpdate` and is calculated
158  from the `fixed_tickrate` constructor parameter; and the
159  fixedUpdateLag(), this last value is the Time since the last `fixedUpdate`.
160 
161  The running state of the Game can be controlled by using setRunning(). If set
162  to `false` the game loop will exit and the Game will end. A Game instance
163  should not be reused once the Game is done running, as the final state is not
164  guaranteed in any way.
165 
166  The Game class also owns the Actor object pool and therefore handles the
167  creation and destruction of Actors. Example code:
168  \code
169  hum::Game game;
170  hum::Actor* new_actor = game.makeActor(); // creation of a new Actor;
171  // ...
172  game.destroy(new_actor); // mark the Actor to be destroyed.
173  \endcode
174 
175  Actor%s are not destroyed right away, but marked to be destroyed and destroyed
176  after `Plugin::postUpdate()`. All Actor%s are destroyed automatically after
177  `Plugin:gameEnd()`.
178 
179  A Game instance can also contain Plugin%s. Plugin%s can implement functionality
180  for the Game such as a rendering pipeline, scene management, etc. They can be
181  added and queried by typename using addPlugin() and getPlugin() template methods
182  respectively (example below).
183 
184  \code
185  class MyPlugin : public hum::Plugin {...};
186 
187  hum::Game game;
188  MyPlugin* mp = game.addPlugin<MyPlugin>();
189 
190  // somewhere else in the code (p.e. inside a Behavior)
191  MyPlugin* mp = game().getPlugin<MyPlugin>();
192  \endcode
193 
194  A Plugin shouldn't be added after calling run().
195 */
196 }

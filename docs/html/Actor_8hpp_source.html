194 /*!
195  \class hum::Actor
196  \brief Actor (also known as GameObject).
197 
198  This class represents an object in the Game. You can create a new Actor by calling
199  Game::makeActor(). This method will create a new Actor and return it. The Game owns
200  the Actor and it controls its lifetime.
201 
202  To __destroy__ an Actor you __must__ call Game::destroy(), not its destructor.
203  The Actor then, will be marked to be destroyed and after the next update step
204  it'll be deleted. Just before being deleted, the Actor will call its
205  Behavior%s onDestroy() method.
206 
207  An Actor, by default, doesn't have any behaviour and __you should not inherit
208  from it__. Instead Actor%s are composed by Behavior%s. These Behavior%s are
209  the ones that must implement the behaviour of the Actor composed by them.
210 
211  On the other hand, Actor%s do have a Transformation, accessible through transform()
212  and a reference to its Game, accessible through game().
213 
214  A Actor can be __active__ or __inactive__. If a Actor is __inactive__ it exists,
215  all its Behavior%s also exist and have been instantiated; but it __won't__
216  be updated. Same applies to its Behavior%s.
217 
218  Usage example. The Actor will be destroyed after 10 fixed updates:
219  \code
220  // We define two Behaviors: A and B.
221  class B : public hum::Behavior
222  {
223  public:
224  B(int x): value(x)
225  {}
226 
227  int value;
228  }
229 
230  class A : public hum::Behavior
231  {
232  public:
233  A(int x): current(x)
234  {}
235 
236  void init() override
237  {
238  last = actor().getBehavior<B>()->value;
239  }
240 
241  void fixedUpdate() override
242  {
243  current++;
244  if (current > last)
245  {
246  actor().game().destroy(actor());
247  }
248  hum::log("Count: ", current);
249  }
250 
251  private:
252  int last, current;
253  }
254 
255  int main()
256  {
257  hum::Game g;
258  hum::Actor* a = g.makeActor();
259  // here we add the A and B to the Actor.
260  A* t = a->addBehavior<A>(1);
261  a->addBehavior<B>(10);
262  g.run();
263  return 0;
264  }
265  \endcode
